# keiba_tool

設計書準拠で構築・運用する競馬ツールです。

## 目的

本プロジェクトは、競馬データの取得・解析・学習・予測を、保守しやすく拡張可能な構造で運用することを目的とします。  
単に「動くコード」を置くのではなく、再利用・保守・共有に耐える開発標準を守ることを前提とします。

---

## 開発前提

- Python 3.12 系を使用
- 仮想環境は `venv`（`.venv`）で統一
- GitHub でソース管理
- ローカル依存を排除し、再現性を最優先とする

---

---

## 各ディレクトリの役割

- `main.py`  
  単一入口です。CLI / GUI の起動分岐のみを行います。ロジックは書きません。

- `config/`  
  パス、アプリ設定、各種定数やフラグを管理します。マジックナンバーやベタ書き設定はここへ集約します。

- `core/logic/`  
  入力検証、処理制御、サービス呼び出し、整形処理を担います。

- `core/services/`  
  外部入出力を担当します。スクレイピング、CSV入出力、モデル保存読込などをここへ集約します。

- `core/utils/`  
  ログ、日付、パス、文字列整形などの共通処理を管理します。

- `core/models/`  
  データ構造、戻り値、集約オブジェクトを定義します。

- `ui/`  
  画面表示と入力受付のみを担当します。ロジックは禁止です。

- `tests/`  
  入力検証、ロジック処理、異常系を中心に最低限のテストを置きます。

- `docs/`  
  要件、設計、運用ルールを保存します。

- `logs/`  
  アプリ実行ログを保存します。

- `data/`  
  取得データ、加工データ、一時データを保存します。

## セットアップ

### 1. 仮想環境の作成

```bash
python -m venv .venv
```

### 2. 仮想環境の有効化

Windows:

```bash
.venv\Scripts\activate
```

### 3. 依存関係のインストール

```bash
pip install -r requirements.txt
```

---

## 起動方法

### GUI モード

```bash
python main.py --mode gui
```

### CLI モード

```bash
python main.py --mode cli
```

---

## 実装ルール

- 1関数1責務
- 例外処理必須
- ログ出力必須
- マジックナンバー禁止
- 型ヒント推奨
- UI にロジックを書かない
- `main.py` に処理を詰め込まない
- 例外を握り潰さない
- 構成を途中で崩さない
- README 未整備で共有しない

---

## GitHub 運用

- `main` : 安定版
- `feature/*` : 機能追加
- `fix/*` : バグ修正

コミット前の必須事項:

- `requirements.txt` 更新確認
- README / docs 更新確認
- 動作確認
- 例外・ログ確認

---

## 注意事項

- 外部サイトの HTML 構造変更時は、パーサや取得処理の修正が必要です。
- 学習処理は、結果データと特徴量生成処理の整合が前提です。
- GUI / CLI は同一ロジックを呼ぶことを厳守してください。
