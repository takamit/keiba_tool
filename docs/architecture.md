# Architecture

## 1. 設計方針

本プロジェクトは、UI とロジックを完全に分離し、外部入出力を services 層に集約し、再利用可能な logic 層を中心に構成する。  
CLI と GUI は入口に過ぎず、同一の処理本体を呼び出す設計を厳守する。

---

## 2. レイヤー責務

### main
責務:
- 起動入口
- CLI / GUI の分岐
- 初期設定読込
- ログ初期化

禁止:
- 業務ロジック
- スクレイピング処理
- データ整形
- ファイル処理

---

### ui
責務:
- 入力受付
- 表示
- 実行結果表示
- ユーザー向けエラーメッセージ表示

禁止:
- 業務ロジック
- 特徴量生成
- モデル学習
- 外部通信
- ファイル保存本体

---

### core/logic
責務:
- 入力検証
- 処理制御
- サービス呼び出し
- 表示用データ整形
- 学習・予測フロー制御

想定例:
- レース検索ロジック
- 取得フロー制御
- 特徴量生成制御
- 予測スコア集約

---

### core/services
責務:
- 外部入出力
- HTML取得
- スクレイピング
- CSV読込 / 保存
- モデル保存 / 読込
- キャッシュ操作

想定例:
- netkeiba取得サービス
- HTMLキャッシュサービス
- CSVサービス
- モデルサービス

---

### core/utils
責務:
- ログ
- 日付処理
- パス処理
- テキスト正規化
- 共通バリデーション

---

### core/models
責務:
- データ構造定義
- 戻り値オブジェクト
- 学習結果・予測結果の表現

---

## 3. データフロー

標準フローは以下の通りとする。

```text
UI / CLI
  ↓
core.logic
  ↓
core.services
  ↓
外部リソース（Web / CSV / Model / Cache）
```

結果は必要に応じて `core.models` に載せて UI へ返す。  
共通処理は `core.utils` を利用する。

---

## 4. main.py の原則

`main.py` は単一入口である。  
行うべきことは以下に限定する。

- 引数解析
- 起動モード判定
- ログ初期化
- GUI / CLI 起動呼び出し

それ以外の処理を `main.py` に追加してはならない。

---

## 5. CLI / GUI 共通原則

CLI と GUI は同一の logic を呼び出す。  
同じ処理を UI ごとに重複実装してはならない。

### 例
- GUI の「取得」ボタン
- CLI の「取得」コマンド

どちらも同じ `collect_logic` を呼ぶこと。

---

## 6. 命名規則

- ファイル名: `snake_case`
- クラス名: `PascalCase`
- 関数名 / 変数名: `snake_case`
- 定数名: `UPPER_SNAKE_CASE`

禁止:
- 曖昧命名 (`run`, `do`, `process`, `exec` など)
- 同義ディレクトリの混在 (`app`, `lib`, `module` など)

---

## 7. エラー処理方針

禁止:
- `except Exception: pass`
- 無言失敗
- UI だけで握り潰す処理

原則:
- 詳細はログへ残す
- UI にはユーザー向けメッセージを表示する
- 例外は発生箇所に近い層で補足し、必要に応じて上位へ伝播する

---

## 8. ログ方針

- 実行開始 / 終了
- 外部取得開始 / 失敗 / 成功
- 学習開始 / 終了
- 予測開始 / 終了
- 例外発生箇所
- 保存先パス

これらを最低限ログに残すこと。

---

## 9. テスト対象

最低限の対象は以下とする。

- 入力検証
- ロジック処理
- 異常系
- データ整形処理

UI の見た目テストは後回しでもよいが、logic の正当性検証は必須とする。

---

## 10. 禁止事項

- UI にロジックを書く
- `main.py` に処理を詰め込む
- 例外を握り潰す
- 一時対応を放置する
- 命名途中変更
- README 未整備で共有する
- ディレクトリ構成を途中で崩す

---

## 11. この設計の目的

この設計は「とりあえず動くコード」を量産するためのものではない。  
目的は以下である。

- 保守できる
- 拡張できる
- 再利用できる
- 共有しても崩れない

開発速度だけを優先して構造を壊すことは、長期的には損失になる。
