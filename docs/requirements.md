# Requirements

## 1. 概要

本ツールは、競馬データの取得、解析、学習、予測、および結果保存を行うためのアプリケーションである。  
利用者は CLI または GUI から操作でき、処理本体は共通ロジックを利用する。

---

## 2. 目的

- 競馬データを取得できること
- 出走表 / 結果データを解析できること
- 学習用特徴量を生成できること
- モデル学習を実行できること
- 予測を実行し、結果を保存できること
- 保守しやすい構造で継続改善できること

---

## 3. 機能要件

### 3.1 データ取得
- 日付や開催条件を指定して対象レースを取得できること
- レースIDを収集できること
- 出走表ページおよび結果ページを取得できること
- 必要に応じて HTML キャッシュを保存・再利用できること

### 3.2 データ解析
- 出走表 HTML から馬情報を抽出できること
- 結果 HTML から着順・払戻・騎手・馬体重など必要情報を抽出できること
- 解析結果を CSV などへ落とし込めること

### 3.3 特徴量生成
- 学習および予測で必要な列を補完・整形できること
- 欠損値処理を行えること
- 特徴量列の定義を一元管理できること

### 3.4 学習
- 結果データから学習データを生成できること
- モデルを学習し保存できること
- 学習サマリを保存できること

### 3.5 予測
- 出走表データと学習済みモデルから予測を実行できること
- 複数スコアを統合したランキングを生成できること
- 予測結果を CSV 出力できること

### 3.6 UI
- GUI からデータ取得 / 学習 / 予測を実行できること
- CLI からも同等の主要処理を実行できること
- ユーザー向けの実行結果とエラーを表示できること

---

## 4. 非機能要件

### 4.1 再現性
- 仮想環境は `.venv` で統一する
- 必要依存は `requirements.txt` で再現できること
- ローカル固有パスに依存しないこと

### 4.2 保守性
- ディレクトリ構成は固定とする
- レイヤー責務を厳守する
- 命名規則を統一する
- ドキュメントを整備する

### 4.3 拡張性
- 取得方法の変更に対応しやすいこと
- 学習モデルの追加に対応しやすいこと
- GUI / CLI の両入口に対応できること

### 4.4 例外耐性
- 外部取得失敗時に原因をログへ残せること
- UI では無言失敗しないこと
- 失敗時の挙動が把握できること

---

## 5. 入力仕様

想定入力:

- 日付
- 開催場
- レース番号
- 取得期間
- 学習対象データ
- 予測対象出走表データ
- 各種設定値

---

## 6. 出力仕様

想定出力:

- 出走表 CSV
- 結果 CSV
- 学習済みモデル
- 学習サマリ JSON
- 予測結果 CSV
- 実行ログ

---

## 7. エラー処理要件

- 例外は必ずログへ出力すること
- UI ではユーザー向け文言で通知すること
- 無言失敗を禁止すること
- `except Exception: pass` を禁止すること

---

## 8. ログ要件

最低限、以下を記録する。

- 実行開始 / 終了
- 取得対象
- 保存先
- 学習対象
- 予測対象
- 失敗箇所
- 例外詳細

---

## 9. テスト要件

最低限以下を対象とする。

- 入力検証
- ロジック処理
- 異常系
- データ整形
- 学習 / 予測の主要関数

---

- `feature/*` `fix/*` ブランチで作業すること
- コミット前に動作確認すること
